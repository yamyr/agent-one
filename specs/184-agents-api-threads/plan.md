## Summary

Enable Mistral Agents API conversation thread persistence across turns by storing `conversation_id` per reasoner instance, switching from `conversations.start()` to `conversations.append()` on subsequent turns. Remove the misleading `# TODO` for training logger (already wired via Loop inheritance). Add a config toggle `agents_api_persist_threads` for debugging/cost control.

## Key Implementation Details

### 1. Conversation Thread Persistence (agents_api.py)

Each of the 3 reasoner classes (`AgentsApiRoverReasoner`, `AgentsApiDroneReasoner`, `AgentsApiStationReasoner`) needs:

- New instance attribute: `self._conversation_id = None` in `__init__`
- Modified `run_turn()`:
  - If `self._conversation_id is None` OR `settings.agents_api_persist_threads is False`:
    - Call `client.beta.conversations.start(agent_id=..., inputs=[...], handoff_execution="client")`
    - If persist enabled: `self._conversation_id = response.conversation_id`
  - Else (conversation_id exists AND persist enabled):
    - Call `client.beta.conversations.append(conversation_id=self._conversation_id, inputs=[...])`
  - Response parsing via `_parse_conversation_response()` is unchanged (same `ConversationResponse` structure)

### 2. Config Toggle (config.py)

- Add `agents_api_persist_threads: bool = True` to `Settings` class
- Auto-reads from `AGENTS_API_PERSIST_THREADS` env var via pydantic-settings

### 3. Training Logger (agents_api.py)

- Remove `# TODO: integrate training logger` comment at line 113
- Training logging already works via Loop class inheritance:
  - `RoverAgentsApiLoop` extends `RoverLoop` whose `tick()` calls `training_logger.log_turn()`
  - Same for Drone and Station loops
- Verify this is true by inspecting parent `tick()` methods (already confirmed in research)

### 4. Conversation ID Lifecycle

- **Created**: First `run_turn()` call → `conversations.start()` → store `response.conversation_id`
- **Reused**: Subsequent `run_turn()` calls → `conversations.append(conversation_id=...)`
- **Cleared**: Simulation reset → new reasoner instances created in `_register_agents()` → fresh `_conversation_id = None`
