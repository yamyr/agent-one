## Summary

Enable rover-to-rover direct messaging by adding a `notify_peer` tool that calls the existing `send_agent_message()` infrastructure. The receiving rover already sees messages via `get_unread_messages()` in its prompt. Add UI visualization with magenta communication lines using the existing `addCommLine()` system. Update rover prompt to encourage peer coordination.

## Key Implementation Details

### 1. notify_peer Tool (agent.py)

New tool definition following NOTIFY_TOOL pattern:
- Parameters: `target_id` (string, required), `message` (string, required)
- Description explains battery cost and valid targets
- Added to ROVER_TOOLS list (available to all rovers)

### 2. Action Handler (world.py)

`_execute_notify_peer(agent_id, agent, params)`:
- Deduct BATTERY_COST_NOTIFY (with storm multiplier)
- Validate target_id: exists, not self, is rover type
- Call `send_agent_message(agent_id, target_id, message)`
- Return `{ok: True, target: target_id, message: message, position: [x,y]}`

Dispatch in `execute_action()`: new `elif name == "notify_peer"` branch.

### 3. Tick Handler (agent.py)

In RoverLoop.tick(), after action execution:
- If action is `notify_peer` and result["ok"]: broadcast WebSocket event `name="peer_message"` with `payload={target, message, position}`
- Save to sender's memory: `"Sent peer message to {target}: {message}"`

### 4. Prompt Update (agent.py)

Add PEER COMMUNICATION section to `_build_context()`:
- List other active rover IDs (filter agents by type != "station" and type != "drone", exclude self)
- Explain `notify_peer(target_id, message)` tool
- Suggest use cases: share rich vein discoveries, warn about hazards, coordinate sectors

### 5. UI Visualization (WorldMap.vue)

- Add `peer: '#cc44cc'` to COMM_COLORS
- In event watcher: `if (ev.name === 'peer_message') addCommLine(ev.source, ev.payload?.target, 'peer')`
- Existing animation system handles the rest (dashed line, traveling dot, 3s fade)
