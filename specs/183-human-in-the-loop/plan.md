# Implementation Plan: Human-in-the-Loop (UiRequest::Confirm)

**Branch**: `183-human-in-the-loop` | **Date**: 2026-03-05 | **Spec**: `specs/183-human-in-the-loop/spec.md`
**Input**: Feature specification from `/specs/183-human-in-the-loop/spec.md`

## Summary

Add a `request_confirm` tool to the rover agent that pauses the rover's loop and emits a `confirm_request` WebSocket event to the UI. A human operator sees a modal with Confirm/Deny buttons and a countdown timer. Their response (or timeout) unblocks the rover via an `asyncio.Event` mechanism in the Host. The rover system prompt guides usage for storm zones, hazard tiles, and low-battery moves.

## Technical Context

**Language/Version**: Python 3.14+ (server), JavaScript/Vue 3 (UI)
**Primary Dependencies**: FastAPI, Pydantic v2, Mistral AI SDK (server); Vue 3, Vite (UI)
**Storage**: In-memory WORLD dict (simulation state), SurrealDB on port 4002 (unchanged)
**Testing**: pytest with in-memory SurrealDB, Vitest for UI
**Target Platform**: Web application (localhost dev)
**Project Type**: Web application (FastAPI + Vue 3)
**Performance Goals**: Confirmation modal must appear within 1 tick (~3s) of request
**Constraints**: One pending confirmation per agent max; timeout defaults to 30s
**Scale/Scope**: Single concurrent simulation, 1-6 agents, 1 human operator
